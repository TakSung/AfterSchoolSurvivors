# Overview
'방과 후 생존'은 간단한 조작으로 몰려드는 적을 쓸어버리는 쾌감과 아이템 선택에 따른 '운빨 성장'의 재미를 제공하는 게임입니다. 플레이어가 10분 동안 '아무 생각 없이 몰입'하고, 게임 오버 후 '이번 판은 운이 좋았네' 또는 '다른 아이템을 조합해볼까?'라는 생각을 갖게 하는 것을 목표로 합니다.

# Core Features
## MVP 기능 범위
- **플레이어**: 캐릭터 1종 (운동부 학생)
  - 마우스 이동, 자동 공격 기능 구현
- **적**: 캐릭터 3종 (국어, 수학, 교장 선생님)
  - 각기 다른 공격 패턴 및 능력치 적용
- **아이템**: 총 7종 (무기 3종, 능력 4종)
  - 레벨업 시스템 및 아이템별 강화 효과 적용
- **시스템**:
  - 아이템 시너지: 특정 아이템 조합 시 추가 효과 발동
  - 난이도 조절: 시간에 따라 적 능력치 및 함정 생성률 증가
  - 함정: 무작위 위치에 생성, 플레이어에게 페널티 부여
- **제외**: 영구 성장 시스템 (MVP 출시 후 다음 업데이트 과제)

## 상세 기능 명세
### 게임 플레이 흐름
1.  **시작**: 플레이어는 운동장 중앙에서 시작.
2.  **진행**: 마우스 커서 방향으로 자동 이동 및 자동 공격.
3.  **레벨업**: 적 처치 후 경험치 획득 → 레벨업 시 2개의 아이템 선택지 제시.
4.  **아이템 획득**: 최대 6개의 아이템 슬롯에 장착. 동일 아이템 획득 시 레벨업.
5.  **게임 오버**: 체력이 0이 되면 게임 종료. 생존 시간 기록.

### 플레이어 캐릭터
- **이름**: 운동부 학생
- **조작**: 마우스 커서를 따라 자동 이동 (멈춤 없음).
- **공격**: 캐릭터가 바라보는 방향으로 자동 공격.

### 적 캐릭터
- **국어 선생님**: 느린 이동, 넓은 범위 공격 (난이도 상승 시 공격력 증가)
- **수학 선생님**: 빠른 이동, 직선 돌진 공격 (난이도 상승 시 이동 속도 증가)
- **교장 선생님**: 보스급, 특정 시간마다 등장 (난이도 상승 시 공격력 및 이동 속도 모두 증가)

### 아이템 시스템
- **아이템 슬롯**: 총 6칸 (한번 획득 시 버리기 불가)
- **레벨업**: 동일 아이템 획득 시 레벨 상승 (최대 5레벨)
- **무기 아이템 (3종)**: 축구공, 농구공, 야구 배트 (각 레벨별 강화 효과 명시)
- **능력 아이템 (4종)**: 축구화, 농구화, 홍삼, 우유 (각 레벨별 강화 효과 명시)
- **아이템 시너지**:
  - (축구공 + 축구화): 축구공 데미지 30% 증가
  - (야구 배트 + 농구화): 점프 착지 시, 야구 배트 자동 1회 휘두르기

### 난이도 및 함정 시스템
- **난이도 곡선**:
  - 적 개체 수는 유지하되, 5분이 지날 때마다 적의 공격 속도/투사체 속도 30%씩 증가.
  - 5분/10분 경과 시 적 능력치(공격력/이동속도) 1.6배/2.0배 순차적 강화.
- **함정 시스템**:
  - 3분 경과 후부터 맵에 '압정' 함정 무작위 생성.
  - 효과: 플레이어만 영향. 밟을 시 3초간 이동 속도 20% 감소.
  - 중첩: 최대 3회까지 중첩 (최대 60% 감소).

# User Experience
- **사용자 여정**: 플레이어는 운동장 중앙에서 시작하여 마우스 커서 방향으로 자동 이동 및 자동 공격을 수행합니다. 적 처치 후 경험치를 획득하여 레벨업 시 2개의 아이템 선택지를 통해 캐릭터를 강화합니다. 최대 6개의 아이템 슬롯에 아이템을 장착하며, 동일 아이템 획득 시 레벨이 상승합니다. 체력이 0이 되면 게임이 종료되고 생존 시간이 기록됩니다.
- **UI/UX 고려사항**: 간단한 조작으로 몰입감을 높이고, 아이템 선택의 재미를 강조합니다. 게임 오버 후에도 재도전 욕구를 불러일으키는 메시지를 제공합니다.

# Technical Architecture
- **개발 언어**: Python 3.13+
- **라이브러리**: Pygame
- **그래픽**: 2D 도트 또는 SVG 벡터 스타일
- **성능 목표**: 최소 40 FPS 유지

# Development Roadmap
## MVP 요구사항
- 캐릭터 1종, 적 3종, 아이템 7종 구현
- 마우스 이동 및 자동 공격, 레벨업 시스템, 아이템 시너지, 난이도 조절, 함정 시스템 구현
## 미래 개선 사항
- 영구 성장 시스템 추가

# Logical Dependency Chain
1.  **기본 게임 루프 구현**: 플레이어 이동, 자동 공격, 적 생성 및 처치, 경험치 획득, 레벨업 시스템.
2.  **아이템 시스템 구현**: 아이템 획득, 슬롯 관리, 레벨업, 무기 및 능력 아이템 효과 적용.
3.  **적 AI 및 난이도 시스템**: 적 공격 패턴, 난이도 곡선, 함정 시스템.
4.  **아이템 시너지 구현**: 특정 아이템 조합 시 추가 효과 적용.
5.  **UI/UX 개선**: 게임 오버 화면, 아이템 선택 UI 등.

# Risks and Mitigations
- **개발자 숙련도**:
  - 복잡한 물리 엔진이나 3D 그래픽 사용 배제
  - Pygame 라이브러리 집중 활용
  - MVP 범위 내 핵심 기능 우선 개발
- **게임 밸런스**:
  - 초기 버전은 핵심 재미 검증에 집중
  - 출시 후 플레이 데이터 기반으로 아이템/난이도 수치 지속 조정
